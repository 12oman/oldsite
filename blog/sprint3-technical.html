Control and Flow Loops

The statements for, if and while are control flow statements in programming. They control the flow of what gets done. If you imagine a network of train tracks the control statements are the sections of track which direct the train to go down a specific track.


The Dom

Continuing train analogys we could describe the DOM or document object model as something like Harry Beck's famous London Underground tube map. Rather than emphasising distance and geographical accuracy, like other maps, Beck based his on the circuit diagrams he drew for his day job; stripping the sprawling Tube network down to a neat diagram of coloured, criss-crossing lines. The DOM, like the tube map is a diagram with a navagational logic. The DOM makes HTML and CSS elements addressable, the tube map made the london stations accessable. Just like the way that an overwhelming tangle of tracks and stations is cleanly represented by a schema of nodes. What that means is that the HTML structure and CSS rule and even the browser window can be 'travelled to' by the code from JavaScript.


Arrays and Objects

Because Javascript is an object oriented language and I have read some Speculative Realism, to me it seemed to imply the object-oriented maxim 'everything that exists is an object. This is kind of true, but the two do not neatly align. Arrays are objects with charactertics well-suited for enumerational functions, like shuffling a deck of cards. Notably, they store information differently from other objects. So there are array objects and object objects. Arrays are like an excel spreadsheet with no header row cells, whereas Objects have key values and property values.

  • Arrays have square brackets [] and values are defined by their position within the array. They can accept all types of enty strings
  • Objects have curly brackets {} and are defined by name values and property values sometimes refered to as key/value pairs. {} JSON
